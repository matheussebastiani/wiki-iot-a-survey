# Middleware

O texto apresenta que o middleware é uma camada de software que esconde os detalhes técnicos das diferentes tecnologias (RFID, sensores, etc.). Isso permite que o programador foque no desenvolvimento da aplicação sem precisar entender cada protocolo específico de hardware.

### A Arquitetura Orientada a Serviços (SOA)
A maioria das soluções modernas utiliza a abordagem SOA, que decompõe sistemas complexos em componentes menores (serviços) bem definidos. Isso facilita o reuso de software e a adaptação rápida às mudanças do mercado.

## As Camadas do Middleware 
O texto detalha uma estrutura padrão dividida em quatro níveis principais:

1. Aplicações (Top): Onde o usuário interage com o sistema. Ela utiliza os serviços do middleware via protocolos web padrão.

2. Composição de Serviços: Cria aplicações complexas combinando serviços individuais. Aqui, não se vê "dispositivos", apenas "serviços" organizados em fluxos de trabalho (workflows).

3. Gerenciamento de Serviços: Responsável por funções básicas como a descoberta de novos objetos, monitoramento de status e configuração. É aqui que reside o "catálogo" de funções de cada objeto.

4. Abstração de Objetos: A camada mais próxima do hardware. Ela harmoniza a comunicação com dispositivos heterogêneos, traduzindo linguagens específicas de cada fabricante para um padrão comum (como Web Services).

## Segurança e Casos Específicos
- Confiança, Privacidade e Segurança: Essencial para evitar que tags RFID ou sensores sejam usados para vigilância indevida. Essas funções geralmente permeiam todas as camadas da arquitetura.

- Abordagens Alternativas: Além do modelo SOA, o texto cita projetos específicos como:

- Fosstrak: Focado exclusivamente em infraestrutura RFID.

- e-SENSE: Focado em Redes de Sensores Sem Fio (WSN) para inteligência ambiental.

- UbiSec&Sens: Focado em segurança e armazenamento de dados em larga escala.
